# 横向对比实验（cot） deepseek

## 提示词
```
你正在为学校选课管理系统获取需求，请使用思维链（Chain-of-Thought）方法，一步步思考并输出需求。在最终输出时，请将思考结果转化为以系统为主语的需求描述，请为每条生成的需求标注类型（功能需求 / 非功能需求 / 约束性需求 / 质量需求），标注格式为【需求类型】：需求内容。最终只需输出这些需求描述即可。
```



## 结果
```
【功能需求】：系统应允许学生登录后浏览当前学期开设的课程相关信息。
【功能需求】：系统应支持学生在选课期间进行课程的选择和调整操作。
【功能需求】：系统应在学生尝试选择存在时间冲突的课程时给予提醒。
【功能需求】：系统应根据课程的先修要求对学生选课资格进行审核。
【功能需求】：系统应为教师提供查看选课学生名单的查看功能。
【功能需求】：系统应允许教务管理员进行课程信息的管理操作。
【功能需求】：系统应在选课结束后为学生生成课表，并支持导出功能。
【非功能需求】：系统应支持较多学生同时在线进行选课操作。
【非功能需求】：系统应保证良好的可用性，在选课高峰期能够正常使用。
【质量需求】：系统界面应简洁直观，确保学生能够顺利完成选课操作。
【质量需求】：系统应确保数据信息的一致性，避免显示错误。
【约束性需求】：系统必须与学校现有的身份认证体系集成使用。
【约束性需求】：系统需符合教育信息系统的相关安全标准。
```
