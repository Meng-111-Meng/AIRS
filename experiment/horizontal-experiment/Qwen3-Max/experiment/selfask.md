# 横向实验（AIRS） deepseek

## 提示词
```
你正在为学校选课管理系统获取需求，请使用自问自答（Self-Ask）方法，通过提问和回答来探索系统需求，最终只需输出需求即可。在最终输出时，请将思考结果转化为以系统为主语的需求描述，请为每条生成的需求标注类型（功能需求 / 非功能需求 / 约束性需求 / 质量需求），标注格式为【需求类型】：需求内容。最终只需输出这些需求描述即可。
```




## 结果
```
【功能需求】：系统应允许学生浏览当前学期开设的所有课程信息，包括课程名称、课程代码、授课教师、上课时间、地点和学分。  
【功能需求】：系统应支持学生登录后选课，并在选课成功后将其加入该课程的学生名单中。  
【功能需求】：系统应支持学生退选已选课程，并同步更新课程余量和学生课表。  
【功能需求】：系统应提供课程冲突检测功能，在学生尝试选择时间冲突的课程时给出提示并阻止选课操作。  
【功能需求】：系统应允许教师查看其所授课程的选课学生名单。  
【功能需求】：系统应支持管理员添加、修改或删除课程信息。  
【功能需求】：系统应在课程容量满员时自动禁止后续学生选课，并提示“课程已满”。  
【功能需求】：系统应提供学生个人课表生成功能，展示其已选课程的时间安排。  
【非功能需求】：系统应在高并发选课高峰期（如选课开始首小时内）支持至少1000名用户同时在线操作，响应时间不超过3秒。  
【非功能需求】：系统应保证学生选课数据的持久性和一致性，防止因系统故障导致选课记录丢失。  
【质量需求】：系统界面应简洁直观，确保学生在3次点击内完成一门课程的选课操作。  
【质量需求】：系统应具备良好的可访问性，支持主流浏览器（Chrome、Firefox、Safari、Edge）正常运行。  
【约束性需求】：系统必须部署在学校内部服务器上，不得使用外部公有云服务存储学生选课数据。  
【约束性需求】：系统开发必须遵循学校信息系统安全规范，所有用户身份需通过统一身份认证平台验证。
```
