## 推演
```
现在，基于以上锚定，开始需求探索。
请你模拟对“本科生”、“任课教师”和“教务管理员”这三类关键用户进行访谈。请分别列出从每类用户那里搜集到的原始反馈。反馈内容应聚焦于他们在实际操作中遇到的具体问题、不便之处、以及他们直接表达的期望。请使用他们可能使用的自然语言来描述，确保反馈是具体、有场景的，而非笼统的概括。

请严格按以下格式输出：
一、 本科生主要反馈：
[问题描述]：例如，“选课系统在早上9点开放时，经常点不动或者直接报错，等能进去的时候，热门的通选课早就没名额了。”
[具体场景与期望]：例如，“我在安排课程表时，需要自己用纸笔记下时间，再手动比对有没有冲突。希望系统能在我尝试选择有冲突的课时，立刻用红色高亮提示我具体跟哪门课冲突了。”
...（继续列出其他反馈）

二、 任课教师主要反馈：
[问题描述]：例如，“课程结束后，我需要从好几个不同的Excel表格里手动汇总、核对选课学生名单和最终成绩单，非常容易出错，也很耗时。”
[具体场景与期望]：例如，“我希望在课程容量满员后，系统能自动关闭选课通道，而不是需要我每天登陆上去手动查看并关闭。”
...（继续列出其他反馈）

三、 教务管理员主要反馈：
[问题描述]：例如，“目前每学期初都需要人工审核大量的课程替代、学分认定申请，这些申请格式不一，通过邮件或纸质提交，整理和审批流程混乱。”
[具体场景与期望]：例如，“我们需要系统能根据教学计划，自动生成不同专业、不同年级学生的推荐选课列表，作为他们的参考，这样可以减少大量无效的咨询工作。”
...（继续列出其他反馈）
```

**结果**
```
---

**一、 本科生主要反馈：**

**[问题描述]**：“选课系统在每学期选课开放的第一分钟就卡死，页面要么白屏，要么显示‘服务器繁忙’，等十几分钟后能进去了，我想上的‘电影鉴赏’、‘人工智能导论’这种热门课早就被抢光了。”

**[具体场景与期望]**：“我希望系统能像一些售票APP一样，提前把我想要的课加入‘心愿单’，开放时自动帮我排队尝试提交，而不是让我疯狂地刷新、点击，纯粹拼手速和网速。”

**[问题描述]**：“现在排课表全靠自己眼力和记性。我得把心仪课程的授课时间、星期几记在纸上或者手机备忘录里，自己一个个比有没有时间重叠，特别麻烦，有次不小心选了两门时间冲突的课，直到上课才发现。”

**[具体场景与期望]**：“我希望系统有一个‘可视化课表’功能。当我点击一门备选课程，它应该能实时显示在我当前课表的哪个时间格子里，如果有冲突，那个格子就立刻变红并提示冲突课程的名称，让我一目了然。”

**[问题描述]**：“很多课程信息只有个名字和学分，其他啥都不知道。我不知道这门课主要讲什么、用什么教材、老师讲课风格怎么样、往年给分情况如何，感觉像在开盲盒。”

**[具体场景与期望]**：“我希望能看到每门课程的详细介绍页，里面能有教学大纲、授课老师简介，最好还有往届学生的匿名评价和评分概况（比如给分区间），帮助我做出更合适的选择。”

**[问题描述]**：“退课、改选的过程太折腾了。如果想退掉一门课，得先下载申请表，打印出来找任课老师签字，再交到教务处，等他们人工处理。经常因为跑流程错过截止时间。”

**[具体场景与期望]**：“我希望能在系统里直接发起退课或课程替换申请，线上流转给老师审批（老师点个同意就行），然后自动同步到我的课表中。关键节点要有短信或App推送通知我进度。”

---

**二、 任课教师主要反馈：**

**[问题描述]**：“课程结束后，我需要从教务系统导出一个选课名单Excel，从课程平台导出另一个在线学习记录表，再把纸质版的考勤表和期末考试成绩手动汇总到一起，才能做出最终成绩单。这个过程繁琐，极易出错，核对一遍要花半天时间。”

**[具体场景与期望]**：“我理想中的系统，能把我这门课的所有相关数据（选课名单、平时作业/测验成绩、考勤、期末考成绩）整合在一个界面上。我可以直接在线上录入或批量导入各项分数，系统自动按我设置的比例计算出总评成绩，并一键提交。”

**[问题描述]**：“课程容量设置是死的。比如我设了80人容量，满员后系统就关停了。但经常有学生退课，空出名额，或者有学生特别想加选，我需要手动去调整，或者告知教务处操作，很被动。”

**[具体场景与期望]**：“我希望我能有灵活的权限管理。比如，我可以为课程设置一个‘等待队列’，满员后学生可以排队。一旦有人退课，系统能自动按顺序通知队列里的学生补选。或者，我可以临时为特定学生（如辅修生）发放一个‘加选授权码’。”

**[问题描述]**：“我想给选我课的所有学生发个课程通知（比如调整上课教室、发布预习材料），没有直接的办法。要么在课程平台发（但有些学生不看），要么就得让教务员帮忙群发邮件，很不方便。”

**[具体场景与期望]**：“系统应该为每门课程提供一个内置的消息中心或通知模块。我作为任课教师，可以一键向所有选课学生发送通知，并确保他们能在系统首页或通过绑定邮箱/手机收到提醒。”

**[问题描述]**：“我很难及时掌握学生的到课情况。现在要么手动点名（耗时），要么用扫码签到（容易被代签）。期末统计出勤率是个头疼事。”

**[具体场景与期望]**：“希望系统能集成一个可靠、防作弊的签到功能，比如学生需要在指定时间、地点用手机App进行蓝牙或GPS验证签到。我能实时查看每节课的到课名单和统计图表，并方便地将出勤数据与成绩关联。”

---

**三、 教务管理员主要反馈：**

**[问题描述]**：“目前每学期初都需要人工审核大量的课程替代、学分认定、跨专业选课申请。这些申请有的交纸质表，有的发邮件，格式五花八门。我们需要手动核对培养方案、对比课程大纲，整理和流转流程非常混乱，容易遗漏或出错。”

**[具体场景与期望]**：“我们希望所有特殊申请都能在系统中线上化、标准化提交。学生在线填写表单，系统能自动关联其培养方案和待替代课程信息。审核时，我们能在一个后台界面集中处理，可以一键调取相关课程资料，在线审批并填写意见，流程全程可追溯。”

**[问题描述]**：“生成各类统计报表太耗时了。比如，领导要一份‘本学期各学院课程选课人数分布报告’，我需要从后台导出原始数据，自己在Excel里做透视表、画图，每次格式都可能调整，重复劳动很多。”

**[具体场景与期望]**：“我们需要一个强大的后台数据看板和报表生成器。可以预置常用的统计模板（如选课率、班级容量使用率、成绩分布等），也能让我们自定义筛选条件和图表类型，然后一键生成并导出规范化的报告。”

**[问题描述]**：“课程安排（排课）是一个巨大的拼图游戏。现在很大程度上依赖人工经验来安排教室和时间，要避免教师时间冲突、教室资源冲突、专业课程时序冲突等，经常排到后面发现冲突，又得全部推倒重来，效率极低。”

**[具体场景与期望]**：“我们希望系统能提供智能排课辅助功能。输入核心约束条件（如教师不可用时间、教室类型要求、课程连贯性要求等）后，系统能自动生成多个排课草案供我们选择和微调，并高亮提示所有冲突点，极大提升排课效率和合理性。”

**[问题描述]**：“学生经常会问‘我下学期该选什么课？’‘我还差多少学分毕业？’这类问题。我们面对海量咨询，重复解答同样的基础性问题，占用了处理复杂事务的时间。”

**[具体场景与期望]**：“系统应该能根据每个学生的专业、已修课程、培养方案要求，自动计算其已获学分和待修学分，并生成一个个性化的‘学业进度仪表盘’和‘推荐选课清单’。让学生能自助查询，从而减少我们的基础咨询量。”
```

**结果**
```
---

系统需求描述清单

一、 本科生用户需求
1. 【非功能需求】：系统在选课高峰期间（如开放选课的第一分钟内）应保持高可用性，支持至少每分钟 X 万次的并发选课请求，避免出现白屏、服务器繁忙报错或长时间无响应。
2. 【功能需求】：系统应提供“选课心愿单”功能，允许学生提前预选课程，并在选课开放后，按优先级或排队顺序自动尝试提交选课请求。
3. 【功能需求】：系统应提供实时、可视化的课程表预览功能。当学生选择或备选一门课程时，系统需立即在课表界面显示其时间位置，并用红色高亮及文字提示与其他课程的具体冲突详情。
4. 【功能需求】：系统应为每门课程提供详细的课程信息页面，内容需至少包含：教学大纲、授课教师简介、推荐教材、以及往届学生的匿名课程评价与历史成绩分布概况。
5. 【功能需求】：系统应支持在线退课/换课申请流程。学生可在线发起申请，经任课教师在线审批后，系统自动更新学生课表与课程名额，并将关键状态变更通过推送或短信通知学生。
6. 【质量需求】：系统的用户界面（UI）对本科生而言应直观易用，核心操作（如选课、退课、查课表）在3次点击内完成，并符合主流移动端浏览器的兼容性要求。

二、 任课教师用户需求
1. 【功能需求】：系统应提供集成的成绩管理面板，允许教师在一个界面内查看选课学生名单，并在线录入、批量导入或通过API对接方式获取考勤、作业、考试成绩，按预设规则自动计算总评成绩，并支持一键提交至教务系统。
2. 【功能需求】：系统应提供灵活的选课名额管理功能，允许教师为课程设置“等待队列”，并在名额空出时自动按序通知队列学生；同时支持教师生成并分发有限数量的“加选授权码”。
3. 【功能需求】：系统应为每门课程配备课程通知模块，允许任课教师一键向所有选课学生发送通知，并确保通知在系统门户显示，并可选择同步推送至学生绑定邮箱或手机。
4. 【功能需求】：系统应集成可信的课堂签到功能（如基于位置/蓝牙的验证），签到数据自动关联至课程考勤记录，教师可实时查看并导出出勤统计报表。
5. 【质量需求】：系统在处理成绩等敏感数据时，必须保证数据的100%准确性与完整性，任何计算或传输过程均需有防错与审计日志机制。

三、 教务管理员用户需求
1. 【功能需求】：系统应提供标准化的线上特殊申请（如课程替代、学分认定）处理流程，实现申请提交、附件上传、自动关联培养方案、多级审批（教务员、系主任等）线上化，并全程记录审批日志。
2. 【功能需求】：系统后台应提供可自定义的数据看板与报表生成工具，支持教务管理员一键生成预置模板（如选课率、教室使用率）或自定义条件的统计报表，并支持导出为通用格式（如PDF、Excel）。
3. 【功能需求】：系统应提供智能排课辅助引擎，允许管理员输入约束条件（教师时间偏好、教室资源、课程连续性等），由系统自动生成无冲突的排课草案，并高亮显示所有潜在冲突供人工调整。
4. 【功能需求】：系统应为每位学生动态生成“学业进度仪表盘”，直观展示其已修/待修学分、GPA、以及基于培养方案的个性化“推荐选课清单”。
5. 【约束性需求】：系统必须与学校现有的人员认证系统（如LDAP/统一身份认证）、财务系统（如收费接口）及毕业审核系统实现单点登录与数据对接。
6. 【非功能需求】：系统对所有业务数据的操作必须留有完整的审计日志，确保任何培养方案、成绩、课程安排的修改可追溯至具体操作人、时间和内容，以满足教学管理合规性要求。
```
